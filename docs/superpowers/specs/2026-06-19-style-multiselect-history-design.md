# PhotoClub 风格多选与生成历史设计

## 目标

- 风格支持多选，并按用户选择顺序串行执行。
- 生成数量 `n` 对每个选中风格分别生效。
- 风格代表图来自该风格最近一次成功任务的第 `n` 张结果。
- 永久记录风格是否生成过，不因 `output/` 中的原图被移动或删除而改变。
- 提供全选/全部反选和“只显示未生成过”控件。
- 用户照片、历史和派生预览保持本地，不进入 Git。

## 交互设计

风格卡片使用复选框。前端在 `styleIds` 数组中按首次选中的先后保存执行顺序；取消后重新选择的风格追加到末尾。

“全选”按钮有两种行为：当前可见风格未全部选中则按目录显示顺序补选全部可见风格；当前可见风格已经全部选中则反选全部可见风格。筛选开启时，全选只作用于当前可见的未生成风格，避免无意修改隐藏项目。

“只显示未生成过”是显示筛选开关。它不移除已选择但被隐藏的风格。摘要明确显示选中的风格数量和执行顺序。

数量 `n` 表示每个风格生成 `n` 张。例如选中三个风格且数量为二时，服务端创建三个独立 Codex 任务，共生成六张照片。全屏遮罩持续到整批任务全部结束；结果按风格任务完成顺序累计显示。

## 服务端批次

`POST /api/jobs/generate` 接收有序 `styleIds`，验证至少一个且全部存在。服务端为每个风格创建独立任务并按请求顺序加入现有串行队列。每个任务只读取一个风格规则，继续使用相同人物、格式、方向、临时要求和数量。

接口返回批次标识及任务列表。每个任务携带 `batchId`、`styleId` 和批次序号。前端跟踪整个批次而不是单个任务，累积成功结果并显示当前进度。取消操作同时取消当前批次中正在运行和等待中的任务，不影响其他类型任务。

单个风格失败后，后续风格仍继续执行；批次结束时界面保留成功结果并报告失败风格。这样局部失败不会浪费已排队的其他风格。

## 永久历史与代表图

历史存储在 `.control/style-history.json`，键为 `styleId`，记录：

- 最近成功时间；
- 最近任务 ID；
- 最近一次第 `n` 张输出的原始路径；
- 本地代表图相对路径。

任务成功并确认全部输出存在后，以该任务输出数组最后一项作为代表图源，调用本地同步脚本。脚本缩小图片并原子更新 `styles/previews/<styleId>.jpg`，随后原子更新历史。即使原始输出稍后被移动，历史和已复制代表图仍保留。

`.control/` 已被 Git 忽略；新增风格图片忽略规则，所有风格代表图和缩略图均只保存在本地，不提交 Git。当前已被 Git 跟踪的旧风格缩略图停止跟踪，但保留本地文件。`styles/previews/` 通过说明文件保留目录结构；历史和生成照片不上传，脚本与说明文件正常提交。

## 本地同步脚本

新增 `system/tools/syncstylepreview.mjs`，接收风格 ID 和源图片路径。它负责：

1. 校验风格 ID、源文件和目标路径；
2. 使用 macOS 自带图片工具缩放为适合控制台显示的小尺寸 JPEG；
3. 写入临时文件后原子替换 `styles/previews/<styleId>.jpg`；
4. 更新 `.control/style-history.json`。

服务端成功钩子直接复用该脚本导出的函数，命令行入口便于手动修复或重新同步。同步失败会让该风格任务标记失败，避免出现“已生成”但代表图和历史不一致的状态。

## 风格目录与目录接口

风格 Markdown 仍以 `style_id` 和 `name` 为有效性的核心字段，不再要求预制缩略图存在。现有 `thumbnail` 元数据暂时保留以避免批量改写来源文件，但目录加载不再依赖它。

目录接口为每个风格返回 `generated`、`previewUrl` 和最近成功时间。没有历史的风格仍可用，`previewUrl` 为空，界面显示紧凑的无图占位。

媒体路由只允许访问目录扫描确认过的 `styles/previews/<styleId>.jpg`，继续阻止调用方提供任意路径。

## 错误处理

- 未选择风格时拒绝生成。
- `styleIds` 重复时按首次出现去重并保留顺序。
- 任一未知或非法风格 ID 使整个请求在入队前失败。
- 历史文件损坏时目录接口报告结构化问题，但所有风格按未生成显示。
- 代表图同步失败时记录明确错误，不写入成功历史。
- 批次取消时当前任务和同批等待任务均进入取消状态。

## 测试范围

- 目录解析不再要求预制缩略图，并正确合并永久历史和代表图。
- 同步脚本选择输出数组最后一张、缩放、原子更新和拒绝非法路径。
- 服务端按 `styleIds` 顺序创建独立任务，每个任务生成 `n` 张。
- 重复风格去重、空选择和非法选择验证。
- 单任务失败后批次继续，取消只影响当前批次。
- 前端多选顺序、全选/反选、未生成筛选、累计结果和三语文本。
- 完整服务端与现有回归测试。
