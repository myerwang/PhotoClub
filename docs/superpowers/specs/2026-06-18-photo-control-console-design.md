# 照片生成控制台设计规格

## 目标

提供一个本地 HTML 控制台，让用户通过缩略图和明确控件完成以下操作：

- 选择已有的人物多视图结果
- 选择风格
- 选择输出格式
- 输入仅本次有效的额外要求
- 创建独立 Codex 任务生成最终图片
- 从 `input/` 目录列表创建或覆盖人物设定
- 查看任务状态、错误和最终图片
- 打开输出文件夹

启动流程由项目 skill 管理：启动本地服务、使用系统默认浏览器打开页面、监控客户端连接，并在页面关闭后终止相关任务和服务。

## 技术方案

采用无第三方运行时依赖的 Node.js 单进程服务：

- `web/control.html`：单页控制界面
- `web/control.js`：页面状态、租约、心跳、表单和任务状态
- `web/control.css`：响应式工作台样式
- `web/server.mjs`：HTTP API、静态资源、SSE、串行队列、Codex 子进程和生命周期管理
- `system/skills/control/SKILL.md`：启动、健康检查、打开浏览器、停止和故障处理流程

不使用实验性的 Codex App Server 协议。每次生成或人物设定操作都通过 `codex exec` 创建独立任务。

## 界面结构

控制台采用单屏工作台布局。

### 顶部工具栏

- 服务连接状态
- “允许局域网访问”复选框
- “人物设定”按钮
- “打开输出文件夹”按钮

### 生成参数区

1. 人物选择
2. 风格选择
3. 输出格式选择
4. 本次额外要求
5. 生成按钮

### 任务与结果区

- 当前任务类型和参数摘要
- 等待、运行、成功、失败、取消状态
- 结构化阶段日志
- 最终图片预览
- 查看原图
- 打开输出文件夹
- 取消当前任务

生成期间锁定所有会改变任务输入的控件。状态、日志、结果预览和取消按钮保持可用。

## 人物数据规则

- `input/<name>/` 是人物设定的来源目录。
- `profiles/<name>/multiview_reference.png` 是唯一人物结果。
- 不维护“通过”或“失败”业务状态。
- 多视图文件存在且可读取时，该人物可用于最终生成。
- 多视图文件不存在时，该人物不出现在可选人物列表中。
- 人物选择区直接显示最终多视图文件，不创建额外人物缩略图。
- 多视图在固定比例容器内使用 `object-fit: contain`，不得拉伸或裁切。
- 人物设定对话框从 `input/` 的直接子目录读取名称。
- 同名人物重新生成时直接覆盖 `profiles/<name>/multiview_reference.png`。
- 人物设定任务仍必须使用目录内全部可用照片，并输出一张标准胸像 `2 x 3` 多视图。

## 风格数据规则

每个风格由同目录 Markdown 和代表图组成。Markdown 必须包含以下 YAML 元数据：

```yaml
---
style_id: sticker
name: 贴纸
thumbnail: sticker.png
---
```

规则：

- `style_id` 与 Markdown 文件名一致。
- `name` 是界面显示名称。
- `thumbnail` 指向 `styles/` 内的同风格代表图。
- 风格代表图使用 7-Eleven L 竖向比例。
- 推荐代表图尺寸为 `420 x 600`，无需达到打印分辨率。
- 代表图只表达风格，不绑定真实人物身份。
- 风格创建或修改流程必须同步生成或更新代表图。
- 缺少元数据、名称、代表图字段或图片文件时，界面显示配置错误并禁止选择该风格。
- 页面使用固定 L 比例容器和 `object-fit: contain`，不得拉伸。

## 输出格式

服务从 `system/rules/output_formats.md` 读取已注册格式并返回给页面。初始支持：

- `jp_711_photo_l_1051x1500`
- `jp_711_photo_2l_1500x2102`

页面不自行推导尺寸。最终任务必须携带明确的 `format_id`。

## 会话参数边界

额外要求只进入当前任务提示词，不写入 `styles/`、人物设定或系统规则。只有明确的风格修改任务才能持久修改风格定义。

## Codex 任务执行

### 任务模型

- 每次按钮操作创建独立 Codex 任务。
- 任务默认串行执行，同一时间最多一个子进程。
- 等待任务可取消。
- 生成期间页面锁定。
- 控制任务优先使用环境中可用的 mini Codex 模型。
- mini 模型不可用时，记录回退原因并使用当前默认 Codex 模型。
- 图片任务必须通过 `image_gen` 执行，并要求 `gpt-image-2` 或更高能力。
- 不允许图像任务静默降级到较旧图像模型。
- 所有任务使用本机已登录的 Codex CLI 身份和内置工具能力。
- 服务不得读取、要求、保存或向子进程显式传递 `OPENAI_API_KEY`。

### 最终图片任务

任务输入包括：

- 选中的 `profiles/<name>/multiview_reference.png`
- 选中的风格 Markdown
- 明确 `format_id`
- 本次额外要求
- 唯一任务 ID 和建议输出文件名

任务提示必须声明原始 `input/` 照片不得用于最终生成。结果写入 `output/` 的直接子文件。

### 人物设定任务

任务输入包括：

- 选中的 `input/<name>/` 目录
- 目录内全部可用图片
- 人物多视图规则
- 固定输出路径 `profiles/<name>/multiview_reference.png`

同名结果直接覆盖。任务完成后页面重新加载人物列表和结果图。

## API 设计

- `GET /api/health`：服务状态和版本
- `GET /api/catalog`：人物、输入目录、风格和输出格式
- `GET /api/state`：租约、队列、当前任务和最近结果
- `POST /api/lease`：获取单客户端租约
- `POST /api/heartbeat`：续租
- `POST /api/release`：页面关闭时主动释放
- `POST /api/network`：切换本机或局域网监听模式
- `POST /api/jobs/generate`：创建最终图片任务
- `POST /api/jobs/profile`：创建或覆盖人物多视图任务
- `DELETE /api/jobs/:id`：取消等待或运行任务
- `GET /api/events`：SSE 状态和日志流
- `POST /api/open-output`：调用系统文件管理器打开 `output/`
- `GET /media/*`：只读提供白名单目录内的图片

所有路径由服务端基于目录扫描生成。客户端不得提交任意文件系统路径。

## 单客户端访问

- 默认仅绑定 `127.0.0.1`。
- 页面复选框可以动态启用局域网模式。
- 局域网模式绑定 `0.0.0.0` 并显示访问地址。
- 首个成功获取租约的浏览器获得控制权。
- 其他客户端只显示“控制台正在被占用”，操作全部禁用。
- 不提供网页抢占功能。
- 服务端提供显式强制释放命令用于异常恢复。
- 租约通过客户端 ID 和启动时随机访问令牌验证。

切换监听地址时服务端安全重启监听器，但保持任务队列和子进程状态。

## 页面关闭与进程生命周期

- 页面正常关闭时通过 `sendBeacon` 调用释放接口。
- 页面每 5 秒发送心跳。
- 连续 30 秒无心跳时判定控制页面已关闭或断线。
- 页面刷新享有 30 秒重连窗口。
- 租约超时后取消等待队列。
- 若 Codex 子进程正在运行，先发送 `SIGTERM`。
- 5 秒后仍未退出则强制终止子进程树。
- 子进程结束后关闭 HTTP 服务并释放端口。
- 服务重启时租约和运行时任务状态清空。

## 浏览器启动

启动 skill 执行以下流程：

1. 检查 Node.js 和 Codex CLI。
2. 启动 `web/server.mjs` 并获取动态端口。
3. 等待 `/api/health` 成功。
4. 使用 macOS `open <url>` 调用系统默认浏览器。
5. 不使用 Codex 内置浏览器。
6. 输出本机地址、局域网状态、进程 ID 和停止命令。

## 错误体系

错误分为：

- 配置错误：风格元数据、缩略图、格式注册或目录结构无效
- 输入错误：人物、风格、格式或提示缺失
- 冲突错误：租约占用、任务运行中、覆盖目标不可写
- 执行错误：Codex CLI、模型、image_gen、任务超时或取消
- 结果错误：目标文件缺失、尺寸错误、图片不可读取
- 系统错误：端口、权限、浏览器打开或文件夹打开失败

页面显示简明中文错误、错误代码、建议操作和可展开的技术日志。服务端日志不得包含访问令牌或图片二进制内容。

## 安全边界

- 默认仅本机访问。
- 局域网访问需要随机令牌。
- 同一时间只有一个控制客户端。
- 仅允许读取项目内白名单目录。
- 仅允许写入 `profiles/<name>/multiview_reference.png` 和 `output/`。
- 人物名称和风格 ID必须通过目录名白名单校验，拒绝路径穿越。
- `open` 操作只能打开固定的 `output/` 目录。
- Codex 子进程使用项目工作目录和受限沙箱，不使用危险绕过参数。

## 测试与验收

### 单元测试

- 风格 YAML 解析和缺失字段错误
- 人物、输入目录和输出格式扫描
- 路径白名单和穿越防护
- 串行队列、取消和状态转换
- 租约获取、续租、超时和释放
- 模型选择与回退日志
- 页面关闭后的进程清理

### 集成测试

- 假 Codex 子进程成功、失败、超时和取消
- 人物设定覆盖固定目标文件
- 最终图片结果出现后自动刷新预览
- 局域网监听切换保持队列状态
- 输出文件夹打开失败时返回结构化错误

### 浏览器测试

- 桌面和移动布局无重叠
- 人物多视图按 `3:2` 容器等比显示
- 风格代表图按 L 比例容器等比显示
- 生成时参数控件锁定
- 第二客户端显示占用状态
- 错误、日志和结果均可读取
- 页面关闭后服务和子进程按规则退出

## 非目标

- 不提供远程公网部署。
- 不提供多用户协作。
- 不在浏览器内直接实现图像模型调用。
- 不使用 API Key 调用任何文本或图像模型。
- 不使用 Codex 内置浏览器作为正式启动目标。
- 不创建人物专用缩略图文件。
- 不把本次额外要求持久写入风格。
