# PhotoClub 常见打印尺寸与自定义尺寸设计

## 目标

- 在现有 7-Eleven L、2L 基础上增加日本和国际常用照片打印尺寸。
- 提供一次性的自定义像素尺寸，不修改正式格式注册表。
- 保持现有纵向、横向生成逻辑和精确像素输出要求。

## 预设尺寸

所有预设采用约 300 DPI 的打印像素标准，纵向列出短边和长边；横向生成时交换两边。

| 分类 | 名称 | 物理尺寸 | 短边 x 长边像素 |
| --- | --- | --- | --- |
| 日本 | 7-Eleven L | 89 x 127 mm | 1051 x 1500 |
| 日本 | DSC | 89 x 119 mm | 1051 x 1406 |
| 日本 | KG | 102 x 152 mm | 1205 x 1795 |
| 日本 | 7-Eleven 2L | 127 x 178 mm | 1500 x 2102 |
| 日本 | 六切 | 203 x 254 mm | 2398 x 3000 |
| 通用 | A4 | 210 x 297 mm | 2480 x 3508 |
| 国际 | 4 x 6 inch | 4 x 6 inch | 1200 x 1800 |
| 国际 | 5 x 7 inch | 5 x 7 inch | 1500 x 2100 |
| 国际 | 8 x 10 inch | 8 x 10 inch | 2400 x 3000 |

A4 只注册一次，不在日本和国际分类中重复显示。现有 7-Eleven L、2L 的格式 ID 和像素保持不变，避免破坏已有任务和测试。

## 自定义尺寸交互

打印格式列表增加“自定义”选项。选中后显示两个数字输入：短边像素和长边像素。

- 每边必须是 `256` 到 `8192` 的整数。
- 短边必须小于或等于长边。
- 总像素不得超过 `40,000,000`。
- 纵向使用 `短边 x 长边`。
- 横向使用 `长边 x 短边`。
- 自定义尺寸仅用于当前生成请求，不保存到 Markdown、浏览器存储或其他用户配置。

切换回预设格式时保留当前输入值以便本次页面会话内再次选择，但请求中不发送无关的自定义尺寸。

## 数据与接口

正式预设继续由 `system/rules/output_formats.md` 管理，并由现有目录解析器返回。

浏览器选择预设时继续发送 `formatId`。选择自定义时发送：

```json
{
  "formatId": "custom",
  "customFormat": {
    "shortEdge": 1200,
    "longEdge": 1800
  }
}
```

服务端不信任浏览器校验。它通过独立纯函数验证整数、范围、边长顺序和总像素，然后根据方向构造内部格式：

```js
{
  id: "custom_1200x1800",
  label: "Custom 1200 x 1800",
  width: 1200,
  height: 1800
}
```

横向时内部 `width` 和 `height` 交换。该内部格式只传给当前任务的提示词构造器，不加入目录或写入磁盘。

## 错误处理

- 自定义模式缺少任一输入：返回 `CUSTOM_FORMAT_REQUIRED`。
- 非整数、超出单边范围、短边大于长边或超过总像素：返回 `CUSTOM_FORMAT_INVALID`。
- 预设模式忽略请求中多余的 `customFormat`，避免临时输入影响正式格式。
- 未知的预设 `formatId` 继续返回现有 `SELECTION_INVALID`。

中日英界面提供一致的字段、限制和错误提示。错误区域使用现有结构化错误体系。

## 输出规则

所有预设和自定义格式都继续执行以下规则：

- 按目标方向原生构图并生成目标宽高。
- 不得先生成另一方向后旋转或裁切。
- 不因格式增加通用白边或安全边距。
- 只有风格规则明确要求时才应用安全边距。
- 若模型结果尺寸不同，任务必须重新生成或后处理到精确尺寸。

## 测试范围

- 注册表解析出全部新增预设及精确尺寸。
- 自定义尺寸纯函数覆盖最小值、最大值、总像素边界和非法输入。
- 服务端覆盖纵向、横向交换、缺失输入、未知预设和预设忽略多余自定义值。
- 提示词包含自定义格式 ID 和最终精确宽高。
- 中日英文案覆盖自定义字段和错误。
- 浏览器验证自定义字段只在自定义模式显示，并在提交前提供即时错误。
- 完整回归测试确认现有 7-Eleven L、2L 行为不变。
