# 同框照相馆使用说明

## 启动控制台

在 `/Users/yohji/photo` 下运行：

```bash
node web/server.mjs --root /Users/yohji/photo --port 0 --open
```

服务会使用 macOS 系统默认浏览器打开控制页面，不使用 Codex 内置浏览器。关闭页面、切换标签页、切换窗口或心跳中断都不会停止服务，也不会取消生成任务。需要停止服务时，点击页面顶部的“关闭服务”并确认，或使用下方停止命令。

强制释放失效页面租约：

```bash
node web/server.mjs --root /Users/yohji/photo --release
```

停止服务：

```bash
node web/server.mjs --root /Users/yohji/photo --stop
```

## 页面操作

1. 从人物多视图中选择一个或多个人物；每张结果会同时使用全部选中人物生成合照。
2. 从代表图中选择风格。
3. 选择已注册的输出格式和照片方向。横向模式会交换画布长短边并要求模型直接按横向构图。
4. 设置生成数量，默认 1，范围 1–20；可填写只对本次任务有效的额外要求。
5. 点击“生成照片”。生成期间参数区锁定，任务状态和取消按钮保持可用。
6. 成功后点击结果缩略图可打开大图预览，也可点击“打开输出文件夹”。

任务排队或生成期间会显示带动态加载效果的全屏遮罩。遮罩内保留“取消任务”按钮，任务成功、失败或取消后自动关闭。

页面右上角可切换局域网监听。系统同一时间只允许一个控制页面持有操作权，其他页面显示占用状态。

页面顶部可在中文、日本語和 English 之间实时切换。切换会立即更新静态控件、人物与风格列表、任务状态、错误提示和结果预览文字，并在浏览器中记住选择；界面语言不会写入人物设定、风格规则或本次生图提示词。

## 人物设定

- `input/<name>/` 的一个直接子目录代表一个人，目录内可放多张参考照片。
- 在“人物设定”中选择目录后，系统把目录内全部可用图片作为同一个人的直接参考。
- 也可切换到“文字描述”：名称可选；留空时由同一个 AI 任务识别真实人物姓名，或为原创虚构人物命名。明确填写同名则直接覆盖。
- 文字描述若指向真实人物，独立任务会先检索该人物的公开图片并作为直接参考；虚构人物则直接按描述创建。
- 成果物只有一张横向 `2 x 3` 胸像多视图：`profiles/<name>/multiview_reference.png`。
- 人物设定没有成功、失败或批准状态。最终多视图文件存在且可读时，该人物才出现在人物列表中。
- 同名人物重新生成时直接覆盖最终多视图文件；生成失败时不创建状态记录。
- 人物设定不包含风格、提示词、文字五官描述或源照片副本。
- 后续照片生成只使用最终多视图作为身份图片，不再次使用 `input/` 原图。

## 风格规则

贡献者唯一可追加的公共内容位于 `styles/`。每个风格由规则和代表图组成：

```text
styles/
  <style>.md
  <thumbnail>.png
```

规则文件必须以以下元数据开头：

```yaml
---
style_id: sticker
name: 贴纸
thumbnail: sticker.png
---
```

代表图使用 7-Eleven L 竖版比例，推荐 `420 x 600`，只用于直观选择风格，不要求打印分辨率，不绑定真实人物。创建或实质修改风格时必须同步生成或更新代表图。缺字段或缺代表图的风格会被停用。

贴纸风格默认 `3 x 5` 共 15 个贴纸岛，外侧安全白边约为画布宽度的 3%。对于人物贴纸，默认在各贴纸岛之间随机变化服装、服装颜色、表情、手势和姿势，同时保持人物身份稳定。该安全边距只属于贴纸生成规则，不改变输出图片尺寸，也不应用到其他风格。

## 输出格式

- `jp_711_photo_l_1051x1500`: 7-Eleven L，`89 x 127 mm`，`1051 x 1500`
- `jp_711_photo_2l_1500x2102`: 7-Eleven 2L，`127 x 178 mm`，`1500 x 2102`

纵向尺寸分别为 `1051 x 1500`、`1500 x 2102`；横向尺寸分别为 `1500 x 1051`、`2102 x 1500`。最终图片必须精确匹配所选格式和方向。方向本身不增加白边或通用安全区；贴纸安全边距仍只由贴纸风格控制。输出文件直接写入 `output/`，不创建结果子目录。

## 模型与安全

- 每次按钮操作创建独立的 `codex exec` 任务，队列严格串行。
- 若配置 `PHOTO_CODEX_MINI_MODEL`，先尝试该 mini 模型；仅在模型不存在时回退到本机默认 Codex 模型。
- 生图必须使用 Codex 内置 `image_gen`，能力不得低于 `gpt-image-2`。
- 模型调用只使用本机已登录的 Codex 身份。系统不读取、不要求、不保存、不传递 `OPENAI_API_KEY`。
- 页面只提交稳定 ID，不能提交任意本地路径。Web 媒体接口不暴露 `input/` 原图。
